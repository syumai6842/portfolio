# ヒーローセクションからの遷移と設計提案

## コンセプト概要

ヒーローセクション（3Dモデルと"DEEPER"テキスト）から、**ゲームUI風の3D空間ナビゲーション**へと遷移する設計。従来の縦スクロールではなく、**オブジェクトを直接操作して各セクションにアクセス**する体験を提供します。

---

## 1. 遷移のトリガー設計

### 1.1 初期状態（ヒーローセクション）
- 3Dモデル（head + body）が中央に表示
- "DEEPER"テキストが前面に表示
- マウス追跡で顔が回転
- 幾何学的背景がアニメーション

### 1.2 遷移の開始方法

#### オプションA: 長押し/ホールド操作（推奨）
- **操作**: 画面を長押し（2秒）またはマウスをクリックしたまま保持
- **視覚的フィードバック**: 
  - 画面周囲に光のリングが広がる
  - "DEEPER"テキストが徐々に透明化
  - 3Dモデルが後ろに下がる（カメラが引く）
- **効果音**: 低い音が徐々に高くなる（蓄積感）

#### オプションB: ドラッグ操作
- **操作**: 画面を下方向にドラッグ/スワイプ
- **視覚的フィードバック**:
  - カメラが下方向に移動
  - 3D空間が広がっていく感覚
  - オブジェクトが下から浮き上がってくる

#### オプションC: ダブルタップ/ダブルクリック
- **操作**: 画面を2回連続でタップ/クリック
- **視覚的フィードバック**:
  - 画面が一瞬暗転
  - 3D空間が展開される

## 2. 遷移後の3D空間設計

### 2.1 カメラワーク
```
初期位置: (0, 7.5, 3.0) → ターゲット: (0, 7.5, 0)
↓ 遷移後
探索位置: (0, 5, 10) → ターゲット: (0, 0, 0)
```

- カメラが後ろに下がり、広い視野を確保
- 3D空間全体が見渡せる位置に移動
- スムーズなGSAPアニメーションで遷移

### 2.2 3D空間内のオブジェクト配置

#### セクション別オブジェクト設計

**WORKS（作品集）**
- **オブジェクト**: 本棚（Bookshelf）
- **配置**: 左側（x: -8, y: 0, z: 0）
- **操作**: 
  - 本をドラッグして引き出す
  - 本をクリック/タップして開く
  - ページをスワイプでめくる
- **視覚**: 複数の本が並んだ本棚、光る本が選択可能であることを示す

**ABOUT（自己紹介）**
- **オブジェクト**: 時計（Clock）
- **配置**: 中央後ろ（x: 0, y: 0, z: -5）
- **操作**:
  - 時計の針をドラッグして回す
  - 長押しで時計が開く（中に情報が表示）
  - マウス追跡で時計が反応
- **視覚**: 大きな時計、針が動く、文字盤に情報が表示

**SKILLS（スキル）**
- **オブジェクト**: 楽器/道具のコレクション（Instruments）
- **配置**: 右側（x: 8, y: 0, z: 0）
- **操作**:
  - 楽器をクリック/タップして音を鳴らす
  - マウス追跡で楽器が反応
  - ドラッグで楽器を演奏する動作
- **視覚**: 複数の楽器が浮遊、ホバーで光る

**CONTACT（お問い合わせ）**
- **オブジェクト**: 手紙/ノート（Letter）
- **配置**: 手前（x: 0, y: 0, z: 5）
- **操作**:
  - 手紙をクリック/タップして開く
  - フォームに文字を書く動作をアニメーション
  - 手紙を折りたたんで送信
- **視覚**: 手紙が浮遊、開くとフォームが表示

### 2.3 オブジェクトの配置図

```
         [時計: ABOUT]
              |
    [本棚: WORKS] -- [3Dモデル] -- [楽器: SKILLS]
              |
         [手紙: CONTACT]
```

## 3. インタラクション設計

### 3.1 カメラ制御

#### マウス/タッチ操作
- **ドラッグ**: カメラを回転（OrbitControls風）
- **ホイール/ピンチ**: ズームイン/アウト
- **右クリック/長押し**: パン（横移動）

#### キーボードショートカット（オプション）
- `W/A/S/D`: カメラを前後左右に移動
- `Q/E`: カメラを上下に移動
- `R`: カメラを初期位置にリセット
- `Space`: オブジェクトを選択/アクティブ化

### 3.2 オブジェクト操作

#### ホバー/フォーカス
- マウスがオブジェクトに近づくと:
  - オブジェクトが光る（グローエフェクト）
  - オブジェクトが少し浮き上がる
  - カーソルが変化（手のアイコンなど）

#### クリック/タップ
- オブジェクトをクリック/タップ:
  - カメラがオブジェクトにズームイン
  - オブジェクトが開く/展開される
  - セクションの詳細情報が表示される

#### ドラッグ/スワイプ
- オブジェクトをドラッグ:
  - オブジェクトが動く（物理シミュレーション）
  - 本を引き出す、時計の針を回すなど

## 4. UI要素の設計

### 4.1 ミニマップ（全体マップ）

#### 位置とデザイン
- **位置**: 画面右下（固定）
- **サイズ**: `clamp(150px, 15vw, 250px)`
- **デザイン**: 
  - 円形または六角形のマップ
  - 現在地を示す点
  - 各オブジェクトの位置を示すアイコン
  - クリックで直接移動可能

#### 機能
- 現在地の表示
- オブジェクトへの直接移動（クリック）
- ズームイン/アウト（オプション）

### 4.2 ナビゲーションヒント

#### 操作ガイド
- **初期表示**: 画面左下に操作ガイドを表示
  - "長押しで探索開始"
  - "ドラッグでカメラを回転"
- **フェードアウト**: 操作開始後3秒で消える

#### 現在地表示
- **位置**: 画面上部中央
- **表示内容**: 現在のセクション名（例: "WORKS"）
- **アニメーション**: セクション移動時にフェードイン/アウト

### 4.3 オーバーレイUI（モーダル/パネル）

#### セクション詳細パネル
- **表示方法**: オブジェクトをクリック/タップ
- **デザイン**: 
  - ゲームのインベントリ画面風
  - 半透明の背景
  - 3D空間の上に浮遊
- **機能**:
  - セクションの詳細情報を表示
  - 複数のパネルを同時に開ける
  - ドラッグで移動可能
  - 閉じるボタンまたは外側クリックで閉じる

## 5. アニメーション設計

### 5.1 遷移アニメーション

#### ヒーロー → 3D空間
```javascript
// GSAPタイムライン
const transition = gsap.timeline();

// 1. DEEPERテキストをフェードアウト
transition.to('#deeper-text', {
  opacity: 0,
  scale: 0.8,
  duration: 0.8,
  ease: 'power2.in'
});

// 2. カメラを後ろに移動
transition.to(cameraData.position, {
  z: 10,
  y: 5,
  duration: 1.5,
  ease: 'power2.inOut'
}, '-=0.5');

// 3. オブジェクトをフェードイン（順番に）
transition.from('.object-bookshelf', {
  opacity: 0,
  y: -50,
  duration: 0.8,
  ease: 'power2.out'
}, '-=0.5');

transition.from('.object-clock', {
  opacity: 0,
  scale: 0.5,
  duration: 0.8,
  ease: 'back.out(1.7)'
}, '-=0.3');

// ... 他のオブジェクトも同様
```

### 5.2 オブジェクトアニメーション

#### 浮遊アニメーション
- 各オブジェクトがゆっくりと上下に浮遊
- 回転アニメーション（時計の針など）
- パルス（光る）アニメーション

#### インタラクションアニメーション
- ホバー: スケールアップ + グロー
- クリック: スケールダウン → バウンス
- ドラッグ: 物理ュレーション（慣性、弾性）

## 6. レスポンシブ対応

### 6.1 モバイル（〜768px）

#### カメラ調整
- カメラをより近くに配置（z: 8 → 6）
- オブジェクトのサイズを小さく（scale: 0.8）
- オブジェクト間の距離を縮める

#### 操作の簡略化
- ドラッグ操作をスワイプに変更
- ピンチズームでカメラを制御
- タップでオブジェクトを選択（長押しは不要）

#### UI調整
- ミニマップを小さく（100px程度）
- 操作ガイドを簡潔に
- パネルを全画面表示

### 6.2 タブレット（769px〜1024px）

#### 中間的な設定
- カメラ位置: z: 9
- オブジェクトサイズ: scale: 0.9
- タッチとマウス操作の両方に対応

### 6.3 デスクトップ（1025px〜）

#### フル機能
- すべての機能を有効化
- キーボードショートカット対応
- マウス操作の細かい制御

## 8. 技術的な実装ポイント

### 8.1 Three.js
- `OrbitControls`でカメラ制御
- `Raycaster`でオブジェクトのクリック判定
- `Group`でオブジェクトを管理
- `AnimationMixer`でアニメーション（必要に応じて）

### 8.2 GSAP
- カメラアニメーション
- オブジェクトのフェードイン/アウト
- パネルの表示/非表示
- スムーズなトランジション

### 8.3 パフォーマンス最適化
- オブジェクトのLOD（Level of Detail）
- フラスタムカリングの活用
- 不要なオブジェクトの非表示
- モバイルでの軽量化

## まとめ

この設計により、従来の縦スクロールWebサイトの固定観念を破り、**ゲームUI風の体験的なインターフェース**を実現します。ユーザーは3D空間内を自由に探索し、オブジェクトを直接操作して情報にアクセスできる、新しいポートフォリオ体験を提供します。
